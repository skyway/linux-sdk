279  // Sets and returns recording state. The old state is moved out
280  // of the video receive stream and returned to the caller, and `state`
281  // is moved in. If the state's callback is set, it will be called with
282  // recordable encoded frames as they arrive.
283  // If `generate_key_frame` is true, the method will generate a key frame.
284  // When the function returns, it's guaranteed that all old callouts
285  // to the returned callback has ceased.
286  // Note: the client should not interpret the returned state's attributes, but
287  // instead treat it as opaque data.
288  virtual RecordingState SetAndGetRecordingState(RecordingState state,
289  bool generate_key_frame) = 0;
